给 17 届新生的编程建议

  • 编程建议
  • 致新生
other

一、勤练

老祖宗说的好：书读百遍其义自见，编程也是一个道理，用的多了，就会理解了。

而练，我把它分为三个层次：

  1. 不管懂不懂，照抄敲一遍。(主要是练习手感)
  2. 理解每一行代码是在干什么，为什么这么写。(增加对代码的理解，最好能默写出来)
  3. 尝试修改代码。(比如输出：Hello World，改成输出：Hello PHP 等等，如果可以尝试修改代码，让代码更简洁)

如果能做到第三点，那么你也就真正掌握了这一部分了，缺少的只是实战

二、多问

古语有云：敏而好学不耻下问，编程也是这样，编程的知识更新换代的很快，有可能比你晚入行的人掌握的知识是你听都不曾听过的，但是却比你掌握的好用，这个时候就需要不耻下问了，达者为先嘛

问，也是分三点问：

  1. 是什么？(连这个东西是什么都不知道就没必要学下去了)
  2. 有啥用？(方便学完后进行实战)
  3. 怎么用？(随着学习的过程会慢慢了解的)

当然，询问也是有技巧的，毕竟除了培训班或者是任课老师，没有人有义务为你解惑，所以，为了不浪费你和别人的时间，请先仔细阅读《提问的智慧》一书

三、心态

编程是个会死很多脑细胞的活，程序员最经常说的两句话就是：“这段代码为什么能运行？”、“这段代码为什么不能运行？”，有时甚至会 因为一个空格导致整个程序崩溃而无法运行 (我见过无数新手犯过，尤其是学 Python 和 HTML 的)，所以此时心态很重要，只有静下心来，才能找到问题所在，进而解决它

四、别贪心

贪心体现在很多地方，比如：今天看新闻，安卓的市场需求大、前景好，那我去学安卓；明天又说：大数据会火，OK 那我去学大数据处理；后天看云计算是未来最主要的运算途径，好的我去看看云计算；大后天，习大大要求推进人工智能的发展，人工智能我来了~

不知道各位小学有没有读过一篇文章，猴子捡西瓜的故事，三心二意只会竹篮打水一场空，不管是什么语言、不管是什么方向，当你学到一定程度时，你就会发现，其实其他的也是一样的，只是称呼不同罢了，追逐热门而小看冷门是新手最不可取的思想。

还有一个方面就是想要全部弄懂，没有任何一个人敢说互联网的东西他全部都懂，每个人都会有自己的侧重点，比如我侧重 WEB 开发，那么我在其他领域就没必要钻研的很深入，在我选定的区域里钻研即可

五、坚持写博客

做个博客吧，把你的学习生涯记录下来，写博客的好处很多，网上有很多文章都是讲编程有什么好处的 (包括我博客也有)，有兴趣可以看看，在这里我推荐一篇：为什么你应该（从现在开始就）写博客。并且写博客的成本并不高，不需要你出钱，完全可以使用 github(世界级的程序员社区) 免费搭建一个，本站有相关的教程，如果有不懂的也欢迎加我 QQ：3400301560 互相交流 (请在申请中注明来意，谢谢合作)

六、工具

如果说程序员是古代的侠客的话(事实上我一直觉得程序员就是虚拟世界里的侠客，知乎里也有程序员的修真小说，有兴趣可以去看看，很有意思)，那么他的开发工具就是侠客手里的武器，所谓工欲善其事必先利其器，一个好的工具能大幅度缩短代码编写的时间和代码调试的时间，所以如何选择适合自己的工具，是一件挺重要的事。

  • 如果你和我一样：喜欢折腾、热爱 DIY 的话，推荐你使用编辑器软件：sublime text 或者 Visual Studio Code，如果不介意打开文件非常慢并且安装个插件还要翻墙，或者喜欢开源软件的话，可以考虑使用 Atom，补充一句：启动速度、文件打开速度：sublime > VS Code > Atom
  • 如果你喜欢已经预置好各种功能的话(如同 Win10 一样)，可以考虑使用 IDE，即集成开发环境，使用这种工具的话，会失去很多 DIY 的乐趣，并且文件较大，但是里面的功能十分强大，有代码提示、智能纠错、断点调试等等，如果使用编辑器的话需要自己安装，由于不同语言的 IDE 并不相同，我就不推荐了

七、学会看英文文档

对于学编程来说，英文并不重要，来来回回就那么几个英语单词，死记都行，但是英文是很有用的，很多资料原版是英文编写的，翻译成中文后会改变意味，所以推荐能看英文的文档就看英文的，其实文档涉及的大多是编程术语，并不是很多，可以边查字典边看，过段时间就会发现其实不用查也能看得懂了
