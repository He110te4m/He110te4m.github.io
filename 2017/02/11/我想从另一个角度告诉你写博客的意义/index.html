我想从另一个角度告诉你写博客的意义

  • Blog
  • 感悟
杂谈

1. 培养兴趣


编程对于那些对技术不感兴趣，但是由于职业倾向而学习的人群，是很难坚持下去的。

因为程序大多数是比较枯燥的，很多时候折腾了大半天，写了无数行代码，也只能写出一个平平常常不怎么炫酷和实用的效果，但偏偏学习的时候需要不断重复这个步骤，解决bug更是需要挑战思维和体力，到最后快崩溃的时候发现可能只是一个标点符号甚至是一个空格在作怪。

我最开始学习 html 和 css 的时候，对程序并无兴趣，所以一直无法理解身边有个热爱编程的朋友，提到编程眼神就放光的那种心情。

直到后来遇到 PHP 和 E…

学习这回事，兴趣起着至关重要的作用，你不能回避它

有人会说没有兴趣也一样可以学的很好，比如什么通宵熬夜，废寝忘食，诸如此类的，这样的例子大有人在，这样有作用，但是不聪明，太过于僵硬。如果有更好的方法，为什么不用更好的呢？

技术博客虽然没有让人对编程和技术产生兴趣的效果，但是如果技术博客能给人带来关注、人气、沟通、反馈、带来其他的一些积极的事物，那也算是一种产生兴趣的方式。

兴趣是可以培养的，而你要做的，就是在培养兴趣之前，用各种方法坚持下去。如果实在培养不出兴趣，就靠其他奖惩机制坚持下去例如为了高薪~毕竟生存也是一种动力

2. HR 与简历


互联网时代，想要了解一个人的方式不再是朝夕相处，或者道听途说(恋人也好，Hr也罢)，尤其是现在，两个小时你可能只能和对方吃一顿饭，简单的聊聊天，就算是查户口似的问问题，也了解不到多少你想了解的内容，而社交平台有着无法逆转的 时间戳 特性，在一条长长的时间线里，你能通过对方的文字，图片，音/视频，了解到对方网上所呈现出来的一切事物，阅读这些内容，一个小时都已经足够了

作为一个技术博客，大家都有了不同的初衷，不同的写作历程，但在 HR 的简历里，一个好的技术博客无疑是有利的证明，因为一个写技术博客的人至少有两点特质：

  1. 愿意整理经验和分享出来
  2. 能把事情讲明白，使读者能够理解。

程序员多数时候往往是不善于沟通的，写文章可以加强自己表达的能力。作为一个 HR，可以很容易从一次面试中了解对方的阶段和状态，何况是一个完整技术博客。当然前提是你的 博客不要只是单纯地翻译或转载，因为那些都是别人的而不是你自己的东西。

3. 强化学习

直白的说，有时候聊天沟通的时候别人都不见得知道你在说什么，而想要把技术教给别人，则 需要对技术了解的十分透彻和良好的表达技巧，技术博客就是记录自己的学习历程以及学习总结的一个大平台，也是对自身知识体系的一个梳理，在表达的过程中，会不断遇到各种问题，而这些问题往往是不能融会贯通而导致的，如果能够教授一个技术问题并且对方也能够容易的接受，那么，对技术的掌握已有十之八九了。

具体的方法就不说了，如果你身边正好有个比你还要小白的同学，或者说论坛里有人抛出了一个你曾接触学习运用过得问题，试着去帮忙解决吧，一开口，你就知道自己的深浅和不足了，而技术博客，观众就是假想的学生，试着去分享，去教授，并且尽量以一种通俗易懂的方式
